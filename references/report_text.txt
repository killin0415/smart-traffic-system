Abstract 
本專題旨在解決現代城市交通⾯臨的壅塞、停⾞位難尋及資訊分散整合困難等問題。我
們設計並實作了⼀套結合多智慧體系統 (Multi-Agent System, MAS)、微服務架構與即
時數據串流技術的智慧交通導航與對話助理平台。
系統核⼼採⽤ Agent 協作架構，透過 Group Chat Manager 協調 Route Agent、
Parking Agent 及 Forecast Agent 等專家智慧體，並整合 YOLO 影像辨識、LSTM/
GRU 流量預測及 RAG (Retrieval-Augmented Generation) 技術。使⽤者可透過⾃然
語⾔與 Chatbot 互動，獲取考量即時流量與預測模型的動態路徑推薦，以及整合即時影
像分析的停⾞位資訊。
技術實作上，本平台運⽤ Kafka 建立事件驅動架構，並以 gRPC 進⾏⾼效內部通訊，搭
配 TimescaleDB 與 Redis 處理時序數據與快取，確保系統在⼤數據流量下的即時性與
穩定性。本研究期能透過創新的軟體架構與 AI 整合，提供更直覺且智慧化的交通解決⽅
案，有效提升駕駛⼈的出⾏效率。

5. 非功能性需求與系統安全 (Technical Constraints & 
Security) 
	 本章節定義了系統在處理⼤量 YOLO 影像數據與多智能體推論時的安全性與穩定
性標準。
5.1.⾝份驗證與授權 (Authentication & Authorization) 
	 系統採⽤ 無狀態 (Stateless) 的安全架構，以⽀撐分散式叢集需求：
• JWT (JSON Web Token)：Mobile Client 登入後，Spring Boot 簽發包含 User 
UUID 與 Role 的 JWT。後續所有 HTTPS 與 gRPC 請求均需攜帶此 Token。
• OAuth2 / OpenID Connect (預留)：架構圖中 Spring Boot server 可作為 
Resource Server，未來若對接外部 CCTV 廠商 API 或 Google 登入時，可快速
擴展 OAuth2 流程。
• 傳輸安全：所有 Mobile 端與伺服器間的通訊強制使⽤ TLS 1.3 (HTTPS) 加密；
內部組件（如 Spring Boot 與 Agents 之間）使⽤帶有 SSL 的 gRPC。
5.2.併發處理與效能 (Concurrency & Performance) 
	 由於 YOLO 每秒產⽣⼤量事件，系統設計以下機制避免阻塞：
• 非同步訊息隊列 (Kafka)：YOLO Node 與後端服務徹底解耦。當流量激增時，
Kafka 作為緩衝區，由 Stream Processor 依據處理能⼒消費數據，確保系統不
會因瞬間⾼負載崩潰 。

5.3.監控與⽇誌 (Observability) 
	 為確保 Multi-agent service 的⾃動化推論可追蹤：
• 分散式鏈路追蹤 (Distributed Tracing)：導入 OpenTelemetry。當 User 發起
路徑請求，系統會記錄從 Spring Boot -> Chat Manager -> Route 
Agent 的完整調⽤鏈路與耗時，便於定位推論延遲。
• 結構化⽇誌 (Centralized Logging)：所有 Agent 與 YOLO Node 的⽇誌統⼀
透過 Kafka 傳輸⾄ ELK/Loki 叢集。特別記錄 YOLO 偵測錯誤率與 Agent 推論
失敗原因。
• 即時監控⾯板：監控 Kafka 堆積量 (Lag) 與 TimescaleDB 的寫入吞吐量，當 
YOLO 數據延遲超過 30 秒時⾃動觸發告警。
6.結論 
本專案為構建⼀個⾼度整合、可擴展且具備「認知推論」能⼒的智慧交通決策平台。透過 
Spring Boot 作為核⼼閘道器 ，系統橋接了前端 Kotlin Multiplatform 的流暢體驗與後
端 YOLO/AI Agents 的強⼤運算⼒ 。
本設計的關鍵價值在於：
維度 技術實作 ⽬的
安全性 JWT + bcrypt 雜湊 保護⽤⼾隱私並確保請求合法性。
可擴展性 Kafka Broker 叢集 ⽀援未來增加更多 YOLO 偵測節點。
低延遲 Redis 緩存 快速回應 Mobile 端對熱點路⼝狀態的查詢。
推論透明度Explainer Agent 讓使⽤者了解 AI 推薦路徑的背後邏輯。

• 時序數據的深度應⽤：利⽤ TimescaleDB 的 Hypertable 技術 ，將 YOLO 產⽣
的瞬時流量數據轉化為具備預測價值的時序資產，⽀撐 Forecast Agent 進⾏精準
的路況預測 。
• 非同步與解耦的穩定性：藉由 Kafka 叢集 緩衝⼤規模的影像監控事件 ，確保系統
在⾼負載路況下仍能維持優異的響應速度與數據完整性。
• 以⼈為本的智能推論：透過 Chat Manager 整合 Route 與 Explainer Agents ，
將艱澀的交通數據轉化為直觀、具備解釋性的導航建議，顯著提升了終端⽤⼾的信
任度。
未來，隨著系統上線與數據積累，本架構可進⼀步導入更複雜的強化學習模型，實現⾃
動化的交通號誌協同控制，為智慧城市（Smart City）的願景提供實質性的技術⽀撐。
